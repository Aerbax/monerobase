DevMeeting 2017-09-03

Submitted by aerbax on Sun, 09/03/2017 - 16:14

Highlights[ ]

  • Most of the devs are prepping for a new release this month.
    • Due to much more hashrate, the intended hardfork date of Sept 21 2017, may be closer to Sept 16 2017.
    • Discussion is ongoing on whether the hard fork block number should be adjusted to something closer to Sept 21st.
  • hyc has implemented SafeSync mode
    • monerod uses "FAST" mode when catching up, and "SAFE" sync mode when the daemon is caught up.
    • These changes will speed up syncing and help to prevent database corruption.
    • Additionally there's a new "salvage" flag that can be used to help fix a corrupted database.
  • knacc has been working on the RuffCT prototyping.
    • Currently waiting on Sarang and Surae to resolve some notation issues
    • After that, they will be able to decide on an appropriate ringsize for RuffCT.
  • SerHack is ready to release the PrestaShop plugin
    • WHMCS, Magento, and Shopify are planned for the future.
  • vtnerd is working to improve the MyMonero experience. There are some database performance issues.
  • Waiting on Dun and Bradstreet(on behalf of Apple) to verify MyMonero for iOS.
  • lessless has some improved API documentation: https://lessless.github.io
  • There are two proposals open for Monero hardware wallets
    • Ledger Firmware port: https://forum.getmonero.org/7/open-tasks/88160/monero-firmware-for-ledger-wallet/
    • Dedicated Monero Hardware Wallet: https://forum.getmonero.org/7/open-tasks/88149/dedicated-monero-hardware-wallet/
  • Rehrar is working on a Monero Project Quarterly Review document.
    • WIP: https://drive.google.com/file/d/0B0vhnWCa9J5QYW1yc2lRNWE2NzA/view?usp=sharing
  • MoneroMooo is working on Luigi's N-1/N MultiSig
  • Next meeting on Sept 17th, 2017.
