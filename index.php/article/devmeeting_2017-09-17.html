DevMeeting 2017-09-17

Submitted by aerbax on Sun, 09/17/2017 - 16:13

Highlights

  • The release this week had more issues than normal due to testing being so close to release. Going forward, the team is going to try to lock down a more rigid schedule for testing and release:
    • Hard forks in March and September
    • Code freezes in Mid-June and Mid-December
    • This gives more opportunity for beta testing and bug fixes prior to the scheduled release.
    • Also opens the window for more "point releases"
  • With regards to the more structured pace mentioned above, larger patches should start to find their way into the main codebase. None of these should require a fork.
    • Sub-Addresses
    • MultiSig (after signoff from the MRL crew - which says that it tentatively looks good, but requires some proofs. ~End of Nov 2017)
    • ZeroMQ
  • More Unit Testing is needed - and is a great place for new developers to start.
  • Luigi1111 is moving from "backup maintainer" to "Lead Maintainer" on both the GUI and the Site. FluffyPony will move to backup maintainer on both.
  • MyMonero GUI
    • The MyMonero Apple D-U-N-S number has still not been received.
    • The MyMonero Google account is ready for their AppStore (but also needs a DUNS #).
    • Both are still being worked on to add some additional stuff (porting iOS code into the JavaScript App)
  • The 2nd MRL roadmap is ready to be proofread: https://drive.google.com/open?id=0ByPYdEm6O54rZGpTbkcwdWMwcW8
