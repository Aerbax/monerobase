DevMeeting 2017-03-12

Submitted by aerbax on Sun, 03/12/2017 - 16:24

Summary[ ]

  • C++11
    • How much C++11 do we adopt
    • How maintable is it for us?
    • Does it add too much complexity?
      • It may, but that may pay off better in the long run.
      • And is it possible to evolve Monero's code into a modular and manageable set of libraries and tools?
      • Starting with a few core libraries seems to be the agreed upon way to go
      • This may be served better by a more in-depth review of what would need to be rewritten
      • This ties in to the next topic...
  • 0.10.2.2 is almost ready for tagging.
    • After these are fixed, automated builds can restart and a release can be finalized.
      • There are still two PR's that need to merge
      • There is a race condition that needs a fix. This may be fixed within a day.
      • "Fluffy blocks" issue.
